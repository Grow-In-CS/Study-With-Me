# UI & UX

## **UI와 UX의 개념 및 차이**

UX 디자인은 ‘User Experience Design’의 약자로, 사용자가 디지털 제품이나 서비스와 상호작용하는 전 과정에서 느끼는 경험을 설계하고 개선하는 데 초점을 둡니다.

사용자가 어떤 목표를 가지고 들어와 어떤 과정을 거쳐 그 목표에 도달하는지, 그 과정이 얼마나 이해하기 쉽고 불편함이 없는지, 감정적으로 만족스러운지까지 포함해 전반적인 사용 맥락을 다루는 영역입니다.

그래서 UX는 공학, 산업/시각 디자인, 서비스 디자인 등과도 맞닿아 있으며 제품·서비스·시스템 전반에서의 사용자 체험을 분석하고 이를 더 나은 방향으로 설계하는 포괄적인 개념으로 이해됩니다.

UI 디자인은 ‘User Interface Design’의 약자로, 사용자가 실제로 마주하게 되는 인터페이스를 시각적·인터랙션 측면에서 구체화하는 작업을 말합니다.

색상, 타이포그래피, 아이콘, 버튼, 레이아웃, 상태 표시 등 화면을 구성하는 요소들을 조합해 보기 좋고 사용하기 쉬운 형태로 만드는 역할을 담당합니다.

다시 말해 UX가 ‘무엇을 경험하게 할 것인가’를 다룬다면, UI는 ‘그 경험을 어떤 형태로 보여줄 것인가’를 다룹니다.

<br/>

### 둘의 관계성

두 영역은 분리된 개념이지만 완전히 독립적이지는 않습니다.

UX가 사용자 요구와 행동 흐름을 정의하면, UI는 그 정의를 눈에 보이고 조작 가능한 형태로 구현합니다.

자전거를 예로 들면 페달 위치, 안장 높이, 핸들 모양처럼 손에 잡히는 물리적 구성은 UI에 가깝고, 실제로 탔을 때의 주행감, 안정성, 피로도, 만족감처럼 사용자에게 남는 전체적 인상은 UX에 가깝습니다.

UX가 선행되지 않으면 UI는 단지 예쁜 화면에 머무를 수밖에 없고, UI가 좋지 않으면 설계한 경험이 실제 사용 단계에서 무너질 수 있습니다.

<br/>

## UI, UX 어느 것이 먼저일까

최근 실무나 문서에서 ‘UI/UX’보다 ‘UX/UI’라는 순서가 더 많이 사용되는 경향이 있습니다.

이는 단순한 표현상의 유행이 아니라, 실제 제품·서비스를 만들 때 사용자 경험(UX)을 먼저 정의하고 그 이후에 인터페이스(UI)를 설계하는 것이 더 합리적이라는 인식이 자리 잡았기 때문입니다.

UX는 서비스의 목적, 타깃 사용자, 사용 시나리오, 문제점, 사용 환경 등을 조사하고 분석해 “이 제품은 사용자가 이렇게 이용하도록 설계돼야 한다”는 방향을 먼저 세웁니다.

사용자 조사, 요구사항 도출, 페르소나 정의, 시나리오 작성, 정보구조(IA) 설계 같은 활동이 이 단계에 포함됩니다.

UI는 이렇게 정의된 경험을 실제 화면으로 떨어뜨리는 단계입니다.

UX 단계에서 나온 사용자 흐름, 와이어프레임, 우선순위 정보 구조를 근거로 버튼 위치를 정하고, 컬러와 컴포넌트를 선택하며, 반응과 전환 애니메이션을 설계해 개발자가 구현할 수 있는 최종 산출물의 형태로 정리합니다.

실무에서 UI가 ‘결과물’처럼 보이는 이유도 이 때문입니다.

앞단의 UX에서 방향이 명확할수록 UI는 일관성 있고 목적에 맞게 제작될 수 있습니다.

이 때문에 본질적으로 두 영역은 관점과 투입 시점이 다릅니다.

UX는 문제를 정의하고 해결 방향을 설계하는 선행 작업이고, UI는 그 설계를 실제로 눈앞에 보이게 만드는 후행 작업입니다.

<br/>

## 좋은 UX/UI란?

UX/UI 디자인에는 실무에서 널리 인정받는 정석적인 법칙과 원칙들이 존재합니다.

이 원칙들은 사용자 경험을 최적화하고, 직관적이고 일관된 인터페이스를 만드는 데 필수적입니다.

<br/>

### **대표적인 UI/UX 디자인 법칙 및 원칙**

- 사용자 중심 디자인: 모든 결정은 실제 사용자의 요구와 관찰, 테스트, 피드백에 기초해야 합니다.
- 직관성과 단순함: 인터페이스는 복잡하거나 부담스럽지 않고, 누구나 쉽게 이해할 수 있어야 하며 주요 동작이 명확히 드러나야 합니다.
- 일관성 유지: 버튼, 컬러, 폰트, 내비게이션 등 디자인 요소의 일관성을 통해 안정감과 신뢰성을 제공해야 합니다.
- 피드백 제공: 사용자의 입력이나 행위에 따라 즉각적이고 명확한 피드백을 주어야 합니다.
- 맥락 있는 흐름과 정보 구조: 사용자가 자연스럽게 목표에 도달할 수 있도록 정보의 우선순위와 흐름을 설계합니다.
- 접근성 고려: 다양한 사용자(연령, 장애 등)가 불편 없이 사용할 수 있도록 글씨 크기, 색상 대비, 시각적 요소를 설계해야 합니다.
- 의미 있는 움직임과 미학적 만족: 애니메이션, 전환 효과 등은 사용자 주의를 흐트러뜨리지 않고 정보 전달과 브랜드 일관성을 높이도록 적용합니다.
- 유연성과 효율성: 반복적 작업은 쉽게 처리하고, 숙련 사용자는 더 빠른 방법(단축키, 설정 등)을 사용할 수 있도록 해야 합니다.
