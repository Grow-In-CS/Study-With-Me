# PWA (Progrssive Web APP)

## PWA란?

프로그레시브 웹 앱(PWA)은 웹 기술을 기반으로 하지만, 네이티브 앱과 유사한 사용자 경험을 제공하는 차세대 웹 애플리케이션 형태입니다.

즉, 웹의 개방성과 접근성을 유지하면서도 앱 수준의 성능, 안정성, 그리고 설치형 환경을 함께 제공하는 기술입니다.

PWA는 웹 앱과 네이티브 앱의 장점을 모두 제공하는, 보다 발전된 형태의 웹 애플리케이션입니다.

기능과 도달범위 측면에서 보면 네이티브 앱은 최고의 기능을 나타내고 웹 앱은 최고의 도달범위를 나타냅니다.

PWA는 네이티브 앱의 기능과 웹 앱의 도달범위가 교차하는 지점에 존재하며, 두 가지 환경의 기능이 모두 포함되어 있습니다.

### 웹

- 연결 가능성
- 기본적으로 액세스 가능
- 편재성
- 손쉬운 배포
- 간편한 업데이트
- 누구나 게시 가능

### 네이티브 앱

- 오프라인 사용 가능
- 고성능
- 기기 통합
- 독립형 환경
- 설치됨 아이콘
- 풍부하고 안정적

요약하자면,

> 웹의 도달성과 앱의 완성도를 결합한 통합형 웹 애플리케이션 기술이 바로 PWA입니다.

<br/>

## PWA의 특징

PWA는 웹과 네이티브 앱의 장점을 모두 결합한 구조를 가집니다.

두 환경이 제공하는 핵심 요소를 다음과 같이 비교할 수 있습니다.

| 구분          | 웹(Web)                   | 네이티브 앱(Native App)   | PWA                         |
| ------------- | ------------------------- | ------------------------- | --------------------------- |
| 접근성        | URL을 통해 즉시 접근 가능 | 앱스토어를 통해 설치 필요 | URL 접근 + 설치 가능        |
| 설치 여부     | 설치 불필요               | 설치 필수                 | 브라우저에서 직접 설치 가능 |
| 업데이트      | 즉시 반영                 | 앱스토어 업데이트 필요    | 자동 업데이트               |
| 오프라인 사용 | 불가                      | 가능                      | 가능 (서비스 워커 기반)     |
| 성능          | 브라우저 의존             | 최적화된 로컬 성능        | 캐싱 기반의 빠른 로딩       |
| 배포 방식     | 서버 배포                 | 스토어 등록               | 서버 배포 + 설치형 지원     |

결과적으로, PWA는 **웹의 유연성과 앱의 안정성을 모두 확보한 사용자 중심의 서비스 제공 방식**입니다.

<br/>

## PWA의 필수 구성 요소

PWA로 인식되기 위해서는 다음의 핵심 기술과 표준 조건을 충족해야 합니다.

1. **HTTPS 환경 (보안성 확보)**

   PWA는 반드시 HTTPS로 제공되어야 합니다.

   모든 데이터 전송이 암호화되어, 보안성이 확보됩니다.

   이는 서비스 워커가 동작하는 기본 전제 조건이기도 합니다.

2. **웹 앱 매니페스트(Web App Manifest)**

   `manifest.json` 파일을 통해 앱의 이름, 아이콘, 시작 URL, 테마 색상 등 메타데이터를 정의합니다.

   브라우저는 이 정보를 기반으로 앱 설치 기능을 활성화합니다.

3. **서비스 워커(Service Worker)**

   브라우저 백그라운드에서 동작하는 스크립트로, **캐싱, 오프라인 사용, 푸시 알림, 백그라운드 동기화** 등을 담당합니다.

   이 기능 덕분에 네트워크가 끊겨도 앱 일부가 정상적으로 동작할 수 있습니다.

4. **반응형 디자인(Responsive Design)**

   데스크톱, 태블릿, 스마트폰 등 다양한 기기에서 동일한 품질의 사용자 경험을 제공해야 합니다.

5. **설치 가능성(Installable)**

   브라우저가 앱 설치를 인식하고, 사용자가 홈 화면 또는 바탕화면에 바로가기를 추가할 수 있어야 합니다.

   설치된 PWA는 독립적인 실행창에서 네이티브 앱처럼 동작합니다.

<br/>

## PWA의 핵심 기능

- **설치 기능**
  브라우저를 통해 사용자가 앱을 직접 설치할 수 있으며, 홈 화면 또는 바탕화면에 아이콘으로 등록됩니다.
  앱 스토어를 거치지 않아도 배포가 가능하므로 배포 비용이 낮습니다.
- **오프라인 동작**
  서비스 워커를 통해 페이지나 리소스를 캐싱함으로써, 네트워크가 불안정하거나 연결이 끊겨도 기본적인 기능을 계속 사용할 수 있습니다.
- **푸시 알림 및 백그라운드 동기화**
  서버에서 전송된 알림을 수신하거나, 연결이 복구되었을 때 자동으로 데이터를 동기화할 수 있습니다.
  이로써 웹 앱에서도 **앱 수준의 사용자 유지 경험**을 제공합니다.
- **자동 업데이트**
  새 버전의 리소스가 배포되면, 서비스 워커가 이를 감지하여 백그라운드에서 업데이트를 수행합니다.
  사용자는 별도 조작 없이 최신 버전을 사용할 수 있습니다.
- **보안성 강화**
  HTTPS 환경을 강제하여 네트워크 공격(중간자 공격, 데이터 탈취 등)을 방지합니다.
- **반응형 사용자 경험**
  디바이스 크기나 OS에 관계없이 최적화된 UI를 제공하며, 네이티브 앱에 가까운 부드러운 인터랙션을 구현합니다.
