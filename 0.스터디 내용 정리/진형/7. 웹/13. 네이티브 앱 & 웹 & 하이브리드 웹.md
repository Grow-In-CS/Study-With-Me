# 📱 네이티브 앱 vs 웹 앱 vs 하이브리드 앱 비교

## 1. 개요

모바일 앱 개발 방식은 크게 **네이티브 앱**, **웹 앱**, **하이브리드 앱**으로 나눌 수 있습니다.  
각 방식은 개발 비용, 성능, 유지보수 난이도, 배포 방식에서 큰 차이를 보입니다.

---

## 2. 비교 요약 표

| 구분 | 네이티브 앱 (Native App) | 웹 앱 (Web App) | 하이브리드 앱 (Hybrid App) |
|------|---------------------------|------------------|------------------------------|
| **개발 방식** | iOS/Android 별로 각각 개발(Swift/Kotlin 등) | 웹 기술(HTML/CSS/JS)로 제작 | 웹 기술 + 네이티브 래퍼(WebView) |
| **성능** | ⭐⭐⭐⭐⭐ 최고 | ⭐⭐ 낮음~보통 | ⭐⭐⭐ 준수 |
| **개발 비용** | 높음 | 가장 낮음 | 중간 |
| **유지보수** | 플랫폼별로 각각 수정 필요 | 서버만 수정하면 즉시 반영 | 중간 |
| **배포 방식** | 앱스토어 등록 필요 | URL 접속 | 앱스토어 등록(선택적) |
| **오프라인 지원** | 좋음 | 제한적 | 중간 |
| **장치 기능 접근** | 매우 쉬움(GPS, 카메라 등) | 제한적 | 대부분 가능 |
| **대표 기술** | Swift, Kotlin | React, Vue, Angular | React Native, Flutter, Ionic |

---

## 3. 상세 설명

### 📱 1) 네이티브 앱 (Native App)
운영체제(OS)에 최적화된 방식으로 개발된 앱.

#### 특징
- 플랫폼별로 **별도 개발 필요** (iOS/Android)
- UI/UX 최적화 및 고성능 앱에 적합
- 점유율 높은 기업(은행·게임·대형 서비스)에서 선호

#### 장점
- 최고 수준의 성능
- OS 최신 기능 활용 가능
- 안정적 오프라인 작동

#### 단점
- 개발비용, 시간 증가
- 두 플랫폼 개발 시 인력 2배 필요

---

### 🌐 2) 웹 앱 (Web App)
웹사이트를 모바일 환경에 최적화하여 앱처럼 보이도록 제작.

#### 특징
- 브라우저 기반으로 실행
- **설치 없이 URL만으로 이용 가능**

#### 장점
- 개발 및 유지보수 비용 가장 저렴
- 배포가 빠르고 쉬움 (수정 = 즉시 반영)

#### 단점
- 성능이 낮음
- 오프라인 기능 제한
- 알림 등 네이티브 기능 접근 제약

---

### 🔀 3) 하이브리드 앱 (Hybrid App)
웹 앱을 기반으로 하되, 네이티브 기술로 감싼 형태(WebView 활용).

#### 특징
- 웹 기술 기반이지만 앱스토어 설치 가능
- 장치 기능도 대부분 사용 가능

#### 장점
- 개발 속도가 빠르고 비용 효율적
- 코드 공유율 높음 (대부분의 기능을 한 번만 개발)

#### 단점
- 네이티브 대비 성능 다소 떨어질 수 있음
- 복잡한 애니메이션/고성능 기능에는 부적합

---

## 4. 어떤 앱 방식을 선택해야 할까?

| 상황 | 추천 방식 |
|------|-----------|
| 고성능, 미션 크리티컬, 은행/게임 수준 | **네이티브 앱** |
| 인력/비용 제한, 빠른 출시 필요 | **웹 앱** |
| 빠른 개발 + 안정적 성능 + 앱스토어 배포 필요 | **하이브리드 앱** |
| 스타트업 MVP 단계 | 웹 앱 또는 하이브리드 |

---

## 5. 결론

- **네이티브 앱** → 최고의 경험과 성능, 하지만 비싸고 오래 걸림
- **웹 앱** → 가장 가볍고 빠르게 출시 가능
- **하이브리드 앱** → 비용과 성능의 균형을 맞춘 선택지
