# 🎨 UI & UX 기본 정리

## 1. UI (User Interface)

UI는 사용자가 제품과 상호작용할 수 있도록 **눈에 보이는 요소들을 설계**하는 분야입니다.

### ✔ UI 핵심 요소
- **레이아웃(Layout)**: 화면 구성 배치
- **타이포그래피(Typography)**: 글꼴, 크기, 두께, 줄 간격
- **컬러(Color System)**: 브랜드 컬러, 대비, 색상 의미
- **아이콘/Iconography)**: 기능을 직관적으로 나타내는 상징 요소
- **컴포넌트(Components)**: 버튼, 입력창, 카드, 모달 등

### ✔ 좋은 UI의 조건
- 일관성(Consistency)
- 명확한 시각적 계층 구조(Visual Hierarchy)
- 접근성(Accessibility)
- 직관적인 조작 가능성(Usability)

---

## 2. UX (User Experience)

UX는 제품을 사용하는 **전체 경험을 설계**하는 분야입니다.

### ✔ UX 핵심 요소
- **사용자 리서치(User Research)**
- **페르소나(Persona)**
- **사용자 시나리오(User Scenario)**
- **UX 플로우(Flow Chart)**
- **사용성 테스트(Usability Test)**

### ✔ 좋은 UX의 기준
- 사용자가 **불편하지 않게** 목표를 달성할 것
- 기능보다 **감정과 만족도** 중심
- 복잡한 요소는 단순화
- 사용자의 행동을 예측한 인터랙션 설계

---

## 3. UI vs UX 차이

| 구분 | UI | UX |
| --- | --- | --- |
| 정의 | 눈에 보이는 화면 요소 | 사용자가 느끼는 전체 경험 |
| 초점 | 디자인, 구성, 배치 | 흐름, 감정, 만족도 |
| 목적 | 보기 좋고 조작 쉬운 화면 | 쓰기 쉽고 계속 쓰고 싶은 서비스 |

---

## 4. UI/UX 디자인 프로세스

1. 문제 정의 (Problem Definition)
2. 사용자 리서치 (User Research)
3. 페르소나 설정
4. 사용자 여정(User Journey) 작성
5. 정보구조(IA) 설계
6. 와이어프레임(Lo-Fi Design) 제작
7. UI 시안(Hi-Fi Design) 제작
8. 프로토타입 제작
9. 사용성 테스트
10. 수정 및 개선(Iteration)

---

## 5. 좋은 UI/UX 예시 원칙

- 클릭해야 할 것은 **크고 명확하게**
- 중요한 정보는 **위쪽 · 굵게 · 큰 글씨로**
- 슬라이더 대신 버튼 조작
- 아이콘 + 텍스트 조합 필수
- ‘생각하게 만들지 않는 UI’ 원칙 준수

---

## 6. UI/UX 체크리스트

### ✔ UI 체크리스트
- 색상 대비 적절?
- 최소 글씨 크기 16px 이상?
- 버튼/터치 영역 충분한가?
- 정렬 및 구성의 일관성 유지?

### ✔ UX 체크리스트
- 사용자가 길을 잃지 않는가?
- 주요 기능까지 클릭 수 적은가?
- 피드백(로딩, 성공/실패) 명확한가?
- 오류 해결 방법 안내 존재?

---

## 📌 결론

UI는 **보이는 것**,  
UX는 **느끼는 것**.

좋은 UI/UX는 기능이 아니라  
사용자가 제품을 **얼마나 편하고 만족스럽게 경험하는가**에 달려 있습니다.
