
# 🌐 Progressive Web App (PWA) 정리

## 1. PWA란 무엇인가?

**PWA(Progressive Web App)**는 웹 기술(HTML, CSS, JavaScript)로 개발하지만,  
네이티브 앱처럼 동작하도록 만든 **하이브리드형 웹 애플리케이션**입니다.

브라우저에서 실행되지만,
- 오프라인 동작
- 푸시 알림
- 홈 화면 설치
- 빠른 로딩 속도  
  등 **앱과 같은 경험**을 제공합니다.

---

## 2. PWA의 핵심 특징

### ✔ 2.1 Service Worker
- 백그라운드에서 동작하는 JS 스크립트
- 캐싱, 오프라인 지원, 푸시 알림 담당
- 네트워크 연결 없이도 앱 실행 가능

### ✔ 2.2 App Manifest
- 웹앱의 설치 정보 정의
- 아이콘, 앱 이름, 화면 표시 방식 등을 지정
- 사용자가 홈 화면에 ‘앱처럼 설치’할 수 있음

### ✔ 2.3 HTTPS 필수
- 보안 강화를 위해 **SSL 적용이 필수**
- Service Worker 기능 활성화 필수 조건

---

## 3. PWA의 장점

| 장점 | 설명 |
|------|------|
| **설치 가능** | 앱스토어 없이 “홈 화면에 추가”로 설치 |
| **오프라인 사용** | Service Worker 캐싱 덕분에 네트워크 없이도 작동 |
| **빠른 성능** | 캐시 기반 로딩 속도 향상 |
| **푸시 알림** | 네이티브 앱 같은 알림 기능 제공 |
| **크로스 플랫폼** | iOS/Android/PC 등 어디서나 동일 코드로 운영 |
| **개발 비용 절감** | 웹 기술 하나로 모든 환경 대응 |

---

## 4. PWA의 단점

| 단점 | 설명 |
|------|------|
| **기능 제한** | 일부 네이티브 API는 사용 불가 (예: 블루투스, Touch ID 등) |
| **앱스토어 노출 약함** | 스토어 기반 마케팅 부족 |
| **iOS 제한** | iOS는 PWA 기능 제한이 상대적으로 많음 (푸시 알림 제한 등) |
| **백그라운드 기능 제약** | 네이티브 대비 백그라운드 동작의 자유도가 낮음 |

---

## 5. PWA 구조 개요

```
📁 Project/
 ├─ index.html
 ├─ manifest.json
 ├─ service-worker.js
 ├─ /assets
 └─ /js
```

- **manifest.json**: 앱 이름/아이콘/색상 정보
- **service-worker.js**: 캐시·오프라인 처리 핵심 로직

---

## 6. PWA의 주요 기능

### 🔹 1. 홈 화면 설치
- 사용자는 웹사이트를 앱처럼 설치 가능
- 풀스크린 모드 지원

### 🔹 2. 오프라인 지원
- 캐시된 화면/데이터 제공
- 네트워크 연결 불안정 환경에서도 안정성 확보

### 🔹 3. 푸시 알림 지원
- 서버에서 메시지를 전송하여 실시간 알림 제공

### 🔹 4. 백그라운드 동기화
- 네트워크 복구 시 자동으로 동기화 처리

---

## 7. PWA VS Native App VS Web App

| 항목 | 웹 앱 | PWA | 네이티브 앱 |
|------|------|------|-------------|
| 설치 | ❌ | ⭕ | ⭕ |
| 오프라인 | ❌ | ⭕ | ⭕ |
| 성능 | 중간 | 중간~높음 | 최고 |
| 개발 비용 | 낮음 | 낮음~중간 | 높음 |
| 앱스토어 | ❌ | 선택 | ⭕ |
| 기기 기능 접근 | 제한적 | 일부 가능 | 거의 전체 가능 |

---

## 8. PWA가 적합한 경우

- MVP를 빠르게 출시하고 싶을 때
- 앱스토어 등록 없이 배포하고 싶을 때
- 유지보수 비용을 줄이고 싶을 때
- 간단한 서비스 중심 앱을 만드는 경우
- 이커머스나 정보 제공 앱

## 10. 결론

PWA는 웹의 접근성과 앱의 성능을 함께 가져가는 **절충형 솔루션**입니다.  
완전한 네이티브 앱은 아니지만, 대부분의 중소형 프로젝트에서  
**비용 대비 최고의 효율**을 제공하며 최근에는 대기업들도 적극 사용 중입니다.
