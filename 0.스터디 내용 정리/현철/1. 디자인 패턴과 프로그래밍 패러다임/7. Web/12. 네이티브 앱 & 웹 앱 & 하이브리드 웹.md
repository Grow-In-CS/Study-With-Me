# 네이티브 앱 & 웹 앱 & 하이브리드 웹

## 네이티브 앱 (Native App)
### 정의
특정 모바일 플랫폼(Android, iOS)에서 플랫폼 전용 언어와 도구를 사용해 개발된 앱.

- Android → Java, Kotlin
- iOS → Swift, Objective-C

### 특징
- 앱 스토어를 통해 배포
- 디바이스 기능(카메라, GPS, 센서 등) 접근 가능
- 설치 후 오프라인에서도 사용 가능

### 장점
- 최고의 성능과 반응속도
- 플랫폼 고유 UI/UX 적용 가능 → 자연스럽고 직관적인 경험 제공
- 복잡한 기능 구현에 유리 (게임, AR/VR 등)

### 단점
- 플랫폼마다 별도 개발 필요 → 개발 비용과 시간이 증가
- 유지보수 및 업데이트 비용 증가
- 앱 스토어 심사 필요

## 웹 앱 (Web App)
### 정의
브라우저에서 실행되는 앱으로, URL 접속만으로 사용 가능하며 설치가 필요 없는 앱.

- HTML, CSS, JavaScript 기반
- 모바일, PC 등 다양한 플랫폼에서 접근 가능

### 특징
- 설치 불필요 → 바로 접속 가능
- 플랫폼 독립적
- 서버에서 업데이트 가능 → 사용자에게 자동 반영

### 장점
- 개발 비용과 유지보수 비용이 상대적으로 낮음
- 한 번 개발하면 다양한 플랫폼에서 사용 가능
- 앱 스토어 심사 필요 없음

### 단점:
- 네이티브 앱만큼 빠르지 않음 → 복잡한 기능 구현에 한계
- 디바이스 기능 접근 제한 (카메라, 블루투스 등)
- 오프라인 사용 제한적

## 하이브리드 앱 (Hybrid App)
### 정의
하나의 코드베이스(예: JavaScript, Dart)로 개발하지만, 실행 시 WebView를 사용하지 않고 각 플랫폼(Android, iOS)의 실제 네이티브 UI 컴포넌트로 직접 변환(컴파일 또는 브릿징)되는 앱.

- HTML, CSS, JavaScript로 개발 후 네이티브 컨테이너 안에서 실행
- React Native, Flutter 등이 대표 기술

### 특징
- 네이티브 앱처럼 앱 스토어를 통해 배포
- 브릿지(Bridge)나 자체 렌더링 엔진(Flutter의 경우)을 통해 대부분의 디바이스 기능(카메라, GPS 등)에 접근할 수 있음
- 하나의 코드로 Android와 iOS 앱을 동시에 개발할 수 있습니다. (코드 재사용성 극대화)

### 장점
- 네이티브 앱에 준하는(Near-Native) 매우 높은 성능과 반응속도를 제공. (WebView 기반 하이브리드 앱 대비 월등히 우수)
- 한 번의 개발로 양대 플랫폼 모두 배포 가능하여 개발 비용과 시간을 획기적으로 절감
- 네이티브 UI 컴포넌트를 직접 사용하거나(React Native) 자체 엔진으로 그려내므로(Flutter) 플랫폼 고유의 UI/UX와 유사한 경험을 제공

### 단점
- OS가 업데이트되어 새로운 기능이 추가될 때, 해당 기능을 지원하는 데 시간이 걸릴 수 있음
- 네이티브 앱 대비 앱의 초기 로딩 속도가 느리거나 파일 크기가 클 수 있음
- 복잡하거나 플랫폼 고유의 기능을 구현해야 할 경우, 여전히 일부 네이티브 코드(Java/Kotlin, Swift/Objective-C) 작성이 필요할 수 있음
