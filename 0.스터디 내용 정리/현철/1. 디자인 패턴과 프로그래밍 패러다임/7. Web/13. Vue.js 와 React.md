# Vue.js & React
## Vue.js

Vue.js는 2014년 Evan You가 개발한 진입 장벽이 낮고 직관적인 프론트엔드 프레임워크.
React보다 늦게 등장했지만 명확한 문법 구조와 쉬운 사용법 덕분에 빠르게 인기 있는 프레임워크로 성장.

### Vue.js의 핵심 특징
- 다양한 프로젝트에 바로 적용 가능: CDN 스크립트 하나로 시작할 수 있을 만큼 간단
- SFC(Single File Component) 구조 사용 → 템플릿(HTML), 로직(JS), 스타일(CSS)이 한 파일에 정리
- 양방향 데이터 바인딩 지원 (v-model)
- 명시적이고 직관적인 디렉티브(v-if, v-for, v-bind 등)

### 장점
- 초보자가 배우기 가장 쉬운 프레임워크
- 템플릿 기반으로 구조가 명확해서 가독성 높음
- 공식 생태계가 잘 정비됨 (Vue Router, Pinia/Vuex, Vite, Nuxt)
- 컴포넌트 간 의존성이 적어 확장성 뛰어남

### 단점
- React에 비해 대형 기업 사용 사례가 적음
- 자유도가 높아 팀 규모가 커질수록 코딩 스타일이 흔들릴 수 있음
- TypeScript 도입 시 React보다 설정이 복잡하다는 의견도 존재

## React란?
React는 Meta(구 Facebook)에서 2013년에 공개한 컴포넌트 기반 UI 라이브러리.
Vue와 달리 프레임워크가 아닌 "라이브러리"로 시작했지만, 거대한 생태계를 기반으로 사실상 프레임워크처럼 사용되는 중.

### React의 핵심 특징
- JSX 기반: HTML과 JavaScript를 하나로 결합
- 단방향 데이터 흐름 → 예측 가능하고 유지보수 쉬움
- 지속적인 성능 최적화 (Fiber, concurrent mode 등)
- Hooks 중심의 현대적 개발 방식 (useState, useEffect, useMemo 등)

### 장점
- 가장 큰 생태계 → 라이브러리·도구·컴포넌트 선택 폭이 넓음
- 대형 서비스에서 검증된 안정성
- 타입스크립트와 궁합이 좋고, 구조적·함수형 개발 패턴에 적합
- 다양한 SSR·SSG 프레임워크 존재 (Next.js, Remix)

### 단점
- JSX 문법은 초보자에게 난이도 상승
- 공식적으로 제공되는 도구가 적어 선택의 시간이 필요함 → “라우팅은 어떤 걸 쓰지?” “상태 관리는 무엇이 좋지?” 같은 고민
- state 관리 구조가 복잡해질 수 있음
