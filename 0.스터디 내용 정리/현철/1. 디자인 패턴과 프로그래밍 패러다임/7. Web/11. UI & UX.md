# UI & UX
디자인을 이야기할 때 가장 자주 등장하는 단어가 UI(User Interface)와 UX(User Experience)다. 두 용어는 함께 언급되는 경우가 많지만, 실제로는 서로 다른 목적을 가진 개념이다.

## UI란 (User Interface)
UI는 사용자가 직접 눈으로 보고 손으로 조작하는 모든 요소를 의미한다.

### 대표적인 UI 요소
- 버튼, 입력창, 드롭다운 메뉴
- 색상, 폰트 스타일, 배치 구조(Layout)
- 아이콘, 이미지, 애니메이션
- 화면 간 이동 구조(Navigation)

즉, UI는 어떻게 보이고 어떻게 작동하는가에 집중한다.

### 좋은 UI의 특징
- 명확한 시각적 위계
  - 중요한 버튼은 더 잘 보이도록 크기·색상을 다르게 한다.
- 일관성 있는 디자인 시스템
  - 버튼 간격, 컬러 팔레트, 그림자 스타일 등 규칙이 일관되면 학습 비용이 줄어든다.
- 사용자의 행동을 유도하는 구성
  - 예를 들어, “회원가입” 버튼을 강조하거나 에러 메시지를 빨간색으로 표시하는 것처럼 UI는 올바른 행동을 도와야 한다.

## UX란 (User Experience)

UX는 사용자가 어떤 경험을 하고, 얼마나 쉽게 목표를 달성하는지를 의미한다. 눈에 보이는 요소뿐 아니라, 서비스 전반에서 느끼는 감정·편리함·만족도까지 포함한다.

### 예시
- 앱을 처음 설치했을 때 온보딩이 이해하기 쉬웠는가?
- 필요한 기능을 찾는 데 오래 걸리지 않았는가?
- 결제 과정이 불편하지 않았는가?
- 오류가 발생하면 어떻게 해결해야 하는지 안내해 주는가?

UX는 “사용자가 사용하는 과정에서 어떤 경험을 겪는가” 전부를 포함한다.

### 좋은 UX의 특징
- 사용자가 원하는 목표를 빠르게 달성할 수 있음
  - 불필요한 화면이나 입력을 최소화한다.
- 예측 가능한 행동 흐름
  - 사용자는 “다음에 어떤 일이 일어날지” 자연스럽게 이해할 수 있어야 한다.
- 문제가 발생했을 때 빠르게 복구 가능함
  - 에러 메시지는 문제의 원인과 해결 방법을 명확하게 안내해야 한다.
- 사용자의 맥락(Context)을 고려
  - 예: 모바일에서는 한 손 조작이 중요하고, 데스크톱은 복잡한 정보 구조를 다룰 수 있다.
